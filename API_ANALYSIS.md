# 天气 API 全面剖析与对比报告

本文档整理了本项目中使用的两个核心天气源——**彩云天气 (Caiyun)** 和 **和风天气 (QWeather)** 的 API 能力、端点及数据细节。

---

## 1. 🌈 彩云天气 (ColorfulClouds) API v2.6

彩云天气以其**分钟级降水预报**和**单接口全量数据**为特色。

### 核心机制

- **请求方式**: 单一聚合接口 (All-in-One)。通过参数控制下发的模块。
- **URL**: `https://api.caiyunapp.com/v2.6/{token}/{lon},{lat}/weather.json`
- **必要参数**: `alert=true` (预警), `dailysteps=15` (15 天预报), `hourlysteps=48` (48 小时预报)。

### 数据能力概览

| 数据模块                     | 字段详情                                                                                                                                                                          | 亮点/备注                               |
| :--------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------- |
| **Realtime**<br>(实时天气)   | • **基础**: 温度, 湿度, 风速/风向, 能见度, 气压<br>• **Skycon**: 天气现象代码 (如 `CLEAR_DAY`, `STORM_RAIN`)<br>• **Air Quality**: AQI (国标/美标), PM2.5, PM10, O3, SO2, NO2, CO | 包含简要的 `life_index` (生活指数)      |
| **Minutely**<br>(分钟级降水) | • **Precipitation**: 未来 2 小时每分钟降水强度<br>• **Probability**: 降水概率<br>• **Description**: 文字描述 (例: "未来两小时无降水")                                             | **核心优势**<br>AI 预测模型，精确到街道 |
| **Hourly**<br>(逐小时预报)   | • **范围**: 未来 48 小时 (可扩展)<br>• **包含**: 温度, Skycon, 风力, 降水强度, 空气质量                                                                                           | 连续性好，适合做趋势图                  |
| **Daily**<br>(逐日预报)      | • **范围**: 未来 15 天<br>• **包含**: 最高/最低温, 白天/夜间 Skycon, 降水强度<br>• **Astro**: 日出/日落时间                                                                       | 包含全天平均湿度和空气质量              |
| **Alert**<br>(预警信息)      | • **内容**: 预警标题, 类型代码, 级别, 发布时间, 详细描述                                                                                                                          | 覆盖全国突发气象预警                    |
| **Life Index**<br>(生活指数) | • **项目**: 紫外线 (UV), 舒适度, 洗车, 穿衣, 感冒风险                                                                                                                             | 通常只有摘要描述，无详细建议文本        |

## 2. 🌀 和风天气 (QWeather) API v7

和风天气（前身为和风天气开源项目）提供**专业气象数据**，接口设计**模块化**，遵循气象行业标准，适合构建复杂的专业应用。

### 核心机制

- **请求方式**: 模块化独立接口 (RESTful)。需要分别请求不同 URL。
- **Host**: `https://api.qweather.com/v7/` (商业/试用) 或 `https://devapi.qweather.com/v7/` (免费)。
- **地理编码**: 必须使用 `Location ID` 或 `lon,lat`。通常先调 GeoAPI 获取 ID。

### 主要 API 端点与能力

#### A. 基础天气

| 端点 (Endpoint)              | 数据详情                                                                                            | 备注                                  |
| :--------------------------- | :-------------------------------------------------------------------------------------------------- | :------------------------------------ |
| `/weather/now`<br>实时天气   | 温度, 体感温度, 天气状况 (Icon/Text), 风向/风力/等级,<br>湿度, 降水量, 气压, 能见度, 云量, 露点温度 | 数据非常全面专业                      |
| `/weather/24h`<br>逐小时预报 | 未来 24 小时逐时预报 (含温湿风压雨云)                                                               |                                       |
| `/weather/7d`<br>逐日预报    | 未来 7 天每日详情 (含日/夜间天气, 日出日落, 月相)                                                   | 免费版限制 3-7 天<br>付费版可达 30 天 |

#### B. 特色服务

| 端点 (Endpoint)                  | 数据详情                                                                                                       | 备注                                         |
| :------------------------------- | :------------------------------------------------------------------------------------------------------------- | :------------------------------------------- |
| `/minutely/5m`<br>分钟级降水     | 未来 2 小时每 5 分钟降水预报 (降水量)                                                                          | 类似彩云，但粒度为 5 分钟                    |
| `/indices/1d`<br>生活指数        | **覆盖极广**: 运动, 洗车, 穿衣, 钓鱼, 紫外线, 旅游, 花粉,<br>舒适度, 感冒, 空气扩散, 空调, 墨镜, 化妆, 晾晒 等 | 含详细的 `text` 建议文本<br>(彩云通常缺这个) |
| `/air/now`<br>空气质量           | AQI, PM2.5, PM10, NO2, SO2, CO, O3, 空气质量等级                                                               | 含监测站数据                                 |
| `/warning/now`<br>灾害预警       | 预警标题, 类型, 等级, 发布时间, 紧迫程度, 确定性                                                               | 标准化预警格式                               |
| `/astronomy/sunmoon`<br>日月升落 | 日出日落, 月出月落, 月相                                                                                       |                                              |

## 3. ⚔️ 综合对比与 Fusion 策略

本项目采用 **Fusion (融合)** 模式，取长补短：

| 特性         | 彩云天气 (Caiyun)                              | 和风天气 (QWeather)                               | **本项目策略 (Fusion)**              |
| :----------- | :--------------------------------------------- | :------------------------------------------------ | :----------------------------------- |
| **地理定位** | 若, 仅支持坐标                                 | **强**, 强大的 GeoAPI (支持中英文城市名/模糊搜索) | **主用 QWeather** 解析地名           |
| **短时降水** | **王者**, 独家 AI 模型, 分钟级文字描述极其精准 | 较好, 提供 5 分钟级数据                           | **主用 彩云** 的分钟级数据和摘要     |
| **生活指数** | 一般, 仅有数值/等级                            | **极佳**, 分类细致, 有贴心的文字建议              | **合并**, 优先用 QWeather 的详细建议 |
| **数据结构** | 聚合式 (一次请求给所有)                        | 分散式 (需多次请求)                               | 后台并行请求，前端统一展示           |
| **空气质量** | 包含在实时接口中                               | 需单独请求                                        | QWeather 为主，缺失时用彩云填充      |

### 总结

- **日常看天**: 和风天气的结构化数据更清晰，图标体系完善。
- **出门避雨**: 彩云天气的 "几点几分下雨/停雨" 描述不可替代。
- **Bot 实现**:
  1. 用 QWeather 找地方。
  2. 用 Caiyun 算下雨。
  3. 用 QWeather 给建议。
  4. 最终呈现一个完美的 "全能气象卡片"。
